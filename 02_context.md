# Prompt para gerar: 01_contexto.md

> Este arquivo será o "briefing permanente" do projeto: o que é, para quem é, qual o objetivo final e quais são as restrições. É o primeiro arquivo a ser gerado, porque os outros dependem dele.

Copie o prompt abaixo, preencha os campos entre [colchetes] e envie para uma LLM.

---

## PROMPT

Você vai me ajudar a criar um arquivo chamado "01_contexto.md", que será o documento de contexto permanente de um projeto que vou desenvolver com a ajuda de uma LLM ao longo de várias conversas futuras.

Informações iniciais que já tenho:
- Nome do projeto: [...]
- Em uma frase, o que é este projeto: [...]
- Por que estou fazendo isso / qual problema resolve: [...]

Antes de escrever o arquivo final, me faça perguntas (pode ser mais de uma rodada) para entender em profundidade:

1. Qual é o objetivo final do projeto e como saberei que ele foi alcançado (critérios de sucesso)
2. Quem é o público-alvo ou para quem este projeto é destinado
3. Quais são as restrições conhecidas (prazo, orçamento, ferramentas, tecnologias obrigatórias, limitações legais/técnicas)
4. Quem são os outros envolvidos (stakeholders, clientes, equipe) e seus papéis
5. Existe algum material de referência, exemplo ou benchmark que devo conhecer
6. Existe algo que definitivamente NÃO deve ser feito (restrições negativas)

Depois das minhas respostas, gere o arquivo 01_contexto.md completo em markdown, com as seções: Visão Geral, Objetivo e Critérios de Sucesso, Público-Alvo, Restrições, Stakeholders, Referências, e O Que Evitar. Seja específico e evite generalidades — use exatamente as informações que eu te dei, sem inventar dados.
